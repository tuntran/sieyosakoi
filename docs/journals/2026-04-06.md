## Decisions

1. **Hardcoded Color Swatches** — Used inline CSS for brand colors instead of CSS variables. Rationale: Colors are fixed (blue, black, red, white) and unlikely to change. Tradeoff: Sacrificed flexibility for simplicity/directness. Could refactor to CSS custom properties if brand expansion occurs.

2. **Static Thumbnail Charts** — Color and size chart images exempt from color-switching logic. Rationale: These are reference images shared across all colors. Implementation: Used `data-static="true"` marker to skip these during color sync.

3. **Server-Side Price Re-pricing** — Checkout re-validates prices from database; never trusts client prices. Rationale: Security against client-side price tampering. Code: `repriced.push()` loop in checkout.astro line 44-52.

4. **Facebook URL Required Field** — Made mandatory for all orders, with normalization (prepend `https://` if missing protocol). Rationale: Needed for future customer outreach. Tradeoff: Adds friction; assumes all customers have Facebook.

5. **Mobile-First Admin Layout** — Hamburger drawer instead of always-visible sidebar. Rationale: Preserve screen space on phones for order details. Drawer auto-closes on navigation.
